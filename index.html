What is CAMARA?

CAMARA is a global initiative (backed by GSMA and Linux Foundation) to create standard APIs for telecom capabilities. Instead of each carrier having proprietary APIs, CAMARA defines one universal interface.

The problem in Canada: Rogers, Bell, and Telus haven't published developer sandboxes yet. If you want to build against these APIs, you have nowhere to test. This sandbox fills that gap with realistic simulation.

What you can test here:

  • SIM Swap Detection — Was a phone number stolen?
  • Number Verification — Does this number match this device?
  • Location Verification — Is this device where it claims?
  • Fraud Scoring — All 3 signals combined into one risk score

How it works

1

Start the sandbox

docker compose up starts the FastAPI simulation server on port 8080. No external dependencies.

2

Use a demo key (or get your own instantly)

Pre-loaded keys for each carrier: demo-sandbox-key-rogers, demo-sandbox-key-bell, demo-sandbox-key-telus. Or call POST /sandbox/keys with your email.

3

Call any CAMARA endpoint

Same endpoints, same request/response format as the real CAMARA spec. Your code works here, it works with real carriers.

4

Get realistic responses

Each carrier has its own latency distribution and error rates (from TOML config). Responses include a _simulation block showing what the engine did.

API surfaces

SIM Swap Detection Live

Detect if a phone number's SIM card was recently swapped — the #1 signal for account takeover fraud.

POST /sim-swap/v1/retrieve-dateWhen was the SIM last swapped?
POST /sim-swap/v1/checkWas it swapped in the last N hours? (boolean)

Number Verification Live

Verify a phone number matches the SIM in a device — no SMS code needed.

POST /number-verification/v1/verifyDoes this number belong to this device?

Location Verification Live

Check if a device is within a geographic area using cell tower data — GPS can be spoofed, cell towers can't.

POST /location-verification/v1/verifyIs this device in the claimed area?

Fraud Score Live Sandbox-only

Combine all 3 signals into a single fraud risk score (0-100). Not part of CAMARA spec — a sandbox convenience.

POST /sandbox/fraud-scoreAll 3 signals → riskScore + riskLevel

First API call in 90 seconds

No carrier agreement. No approval process. No telco knowledge required.

# 1. Clone and start
git clone https://github.com/KachaJugaad/camara-dev
cd camara-dev && docker compose up

# 2. Detect a SIM swap (demo key, no sign-up)
curl -X POST http://localhost:8080/sim-swap/v1/retrieve-date \
  -H "Authorization: Bearer demo-sandbox-key-rogers" \
  -H "Content-Type: application/json" \
  -d '{"phoneNumber": "+14165550100"}'

# 3. Verify a phone number
curl -X POST http://localhost:8080/number-verification/v1/verify \
  -H "Authorization: Bearer demo-sandbox-key-bell" \
  -H "Content-Type: application/json" \
  -d '{"phoneNumber": "+16135550100"}'

# 4. Check device location
curl -X POST http://localhost:8080/location-verification/v1/verify \
  -H "Authorization: Bearer demo-sandbox-key-telus" \
  -H "Content-Type: application/json" \
  -d '{"device":{"phoneNumber":"+16045550100"},"area":{"areaType":"CIRCLE","center":{"latitude":49.28,"longitude":-123.12},"radius":10000}}'

# 5. Get a combined fraud score
curl -X POST http://localhost:8080/sandbox/fraud-score \
  -H "Authorization: Bearer demo-sandbox-key-rogers" \
  -H "Content-Type: application/json" \
  -d '{"phoneNumber":"+14165550100","location":{"latitude":43.65,"longitude":-79.38,"radiusMeters":5000}}'

For Developers Start here

Build fraud detection, identity verification, or location-aware apps against real CAMARA contracts.

1

Start the sandbox

git clone https://github.com/KachaJugaad/camara-dev
cd camara-dev && docker compose up
2

Get an API key

Use a demo key instantly, or create your own:

# Demo keys (no signup needed):
# demo-sandbox-key-rogers | demo-sandbox-key-bell | demo-sandbox-key-telus

# Or create your own:
curl -X POST http://localhost:8080/sandbox/keys \
  -H "Content-Type: application/json" \
  -d '{"email": "you@company.ca"}'
3

Make your first CAMARA call

# SIM swap detection
curl -X POST http://localhost:8080/sim-swap/v1/retrieve-date \
  -H "Authorization: Bearer demo-sandbox-key-rogers" \
  -H "Content-Type: application/json" \
  -d '{"phoneNumber": "+14165550100"}'

# Number verification
curl -X POST http://localhost:8080/number-verification/v1/verify \
  -H "Authorization: Bearer demo-sandbox-key-bell" \
  -d '{"phoneNumber": "+16135550100"}'

# Location verification
curl -X POST http://localhost:8080/location-verification/v1/verify \
  -H "Authorization: Bearer demo-sandbox-key-telus" \
  -d '{"device":{"phoneNumber":"+16045550100"},"area":{"areaType":"CIRCLE","center":{"latitude":49.28,"longitude":-123.12},"radius":10000}}'
4

Use the web playground or CLI

Web UI: Open localhost:3000 — visual playground with SDK snippets
Swagger: Open localhost:8080/docs — interactive API docs
CLI: python scripts/camara-cli.py sim-swap --phone +14165550100

5

Python / Node.js integration

# Python
import httpx
resp = httpx.post("http://localhost:8080/sim-swap/v1/retrieve-date",
    headers={"Authorization": "Bearer demo-sandbox-key-rogers"},
    json={"phoneNumber": "+14165550100"})
print(resp.json())

# Node.js
const resp = await fetch("http://localhost:8080/sim-swap/v1/retrieve-date", {
  method: "POST",
  headers: { Authorization: "Bearer demo-sandbox-key-rogers",
             "Content-Type": "application/json" },
  body: JSON.stringify({ phoneNumber: "+14165550100" }),
});
console.log(await resp.json());

For Operators (Carriers)

Register your real CAMARA endpoint through an 8-step guided wizard. No manual config files needed.

1

Open the operator portal

docker compose up then open localhost:3001

2

Complete the 8-step wizard

Account → MSISDN ranges → Latency profile → Error rates → Feature flags → Endpoint URL → Test → Publish

3

Or use the API directly

# Step 1: Register
curl -X POST http://localhost:8080/operator/register \
  -H "Content-Type: application/json" \
  -d '{"organizationName":"Rogers","contactEmail":"eng@rogers.com","carrierName":"rogers"}'

# Step 2: Configure MSISDN ranges
curl -X POST http://localhost:8080/operator/step/{operator-id}/2 \
  -d '{"msisdnPrefixes":"1416,1647,1437"}'

# Check progress
curl http://localhost:8080/operator/status/{operator-id}
4

Register your real endpoint (optional)

In step 6, provide your real CAMARA endpoint URL. The sandbox will passthrough to it instead of simulating. Run conformance tests in step 7 to validate.
